C'est un débat intéressant et important. Qui va affecter pas seulement la technologie que vous allez utiliser, mais également. Si une société se pose cette question, elle a déjà tout gagner. Elle sait qu'une application optimisée pour le mobile est très important pour son buisiness.

Application Native

Avantages

  • Adaptabilité parfaite avec l'OS
  • Meilleur performance
  • Monétisation de l'application
  • Utilisation de language utilisé en entreprise
  • Meilleur controle de l'Interface Utilisateur
  • Moins d'accès Réseau

Application Native

Inconvénients

  • Développement spécifique pour chaque OS
  • Restriction de contenu
  • Obligation d'Installation, et de mise à jour
  • Cout de développement élevé
  • Différentes versions d'une même technologie
  • Des eco-systèmes moins importants
Pour les couts élévés on peut parler également de l'obligation d'avoir un MAC pour développer.

Application WEB

Avantages

  • Un seul Développement
  • Un déploiement immédiat pour toutes les plateformes
  • Développement HTML5/JS/CSS
  • Indépendant des Stores
  • Des économies sur le coût de développement
  • Très grande communauté derrière les technos WEB
  • Persistence des données
  • Mise à jour facile

Application WEB

Inconvénients

  • Performance
  • Accès au Système
  • Spécification de l'HTML5 très longue
  • UI moins "sexy"
  • Peu de visibilité sur les Stores
  • Trop de Frameworks JavaScript
  • Compatibilité avec les différents navigateurs

2 versions spécifiques

Au départ, les webdesigners créaient 2 versions de leur site web. une pour le desktop et une pour le mobile. Mais avec l'émergence des terminaux connecté, cela serai vite devenu ingérable.

Le Responsive Design

Le Responsive design indique un style de développement d'application web, où le contenu est le même sur chaque appareils, seul l'affichage diffère. Ce terme a été popularisé par un célèbre Web Designer Ethan Marcotte. On peut par exemple retrouver cette idées en architecture. On peut adapter la taille des textes, des images,d es vidéo, la largeur des contenus

Conception d'un design Responsive

  • Construction d'une Grille Fluide
  • Recherche des points de rupture
  • Ecriture des styles qui corrigent les points des ruptures
  • Activation de ces styles en fonction de conditions.

Les Grilles Fluides

  • Utiliser des tailles relatives
  • Utiliser les blocs flottants, permettant un agencement libre
  • ne pas oublier l'alignement vertical (polices)

Media Types

Spécifier le rendu d'un document en fonction d'un média

Un peu d'histoire (au temps du HTML4 et CSS2)

@media print{
	.menu {
		display: none;

Le problème avec cette version c'est la possibilité d'avoir plusieurs types de médias avec des capacités d'affichage différent. De plus, ce fonctionnement dépend beaucoup de son interprétation par les navigateurs. En effet, rien n'oblige un navigateur de choisir la feuille de style qui lui semble la plus approprié pour le device. Par example, il existe un média pour les mobiles qui est ignoré par les plupart des navigateurs mobile.

Pour y remédier, nous allons utiliser les Média Queries, disponible dans les spécifications de CSS3. Les média queries offrent un panel de critères plus vaste et plus précis que les médias présent en CSS2. L'une de confusions qu enos avons concernant le Responsive Design, c'est que nous pensons que le Responsive Design et les Mesia Queries c'est la même chose. En fait le Responsive Design est un ensemble de techniques CSS que nous allons voir en détails. Une média query retourne vrai ou faux. Il suffit donc d'associer les différentes déclarations possibles avec un opérateur logique pour définir l'ensemble des conditions à réunir

  • scan : type de balayage des téléviseurs (progressive ou interlace)

Et les applications Hybrides ?

Qu'est-ce qu'une application hybride ? Une application hybrude est développée avec les même technologies que les sites webs, mais exécutée par un composant natif du mobile. Elles utilisents les composants UIWebView d'iOS ou WebView d'Android par exemple. Comme la plupart des mobile utilise le moteur de rendu WebKit (iOS, Android, Blackberry ...), vous être quasi assuré d'avoir une fonctionnel identique sur toutes les platformes.

PhoneGap est un framework open-source de développement mobile dévolé par Nitobi Software (racheté par Adobe Systems).

	alert(error.message);
